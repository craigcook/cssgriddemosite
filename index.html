Mozilla on CSS Grid

CSS Grid

CSS Grid is a new toolset built directly into web browsers. Grid makes it possible to do innovative page layouts using CSS, without needing heavy frameworks. Grid is a layout framework.

CSS Grid shipped in Firefox 52, on March 7, 2017.

It's coming soon to Chrome 57, on March 14. And to Safari 10.1, in late March or early April.

Example 1

Explain how Grid makes common design ideas from 2008-2016 possible with code that's very easy to write by hand. No more need for 3rd-party layout frameworks.

Here's the heart of the code running this example. The images are part of an HTML list.

<ul class="gallery">
  <li><img src="image1.jpg"></li>
  <li><img src="image2.jpg"></li>
  <li><img src="image3.jpg"></li>
</ul>

The layout is accomplished with incredibly simple CSS. Three lines total accomplish the entire layout.

ul.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
  grid-gap: 1rem;
}

How is this possible? CSS Grid automatically places all of the direct children of a grid container, putting each one in the next cell on the grid. It takes the first list item and puts it in the first box. Then it takes the second item and puts it in the second box. Magic. This is called the Grid Auto-placement Algorithm. It makes coding the layouts we've been designing for years much easier.

And it's also responsive right out the box. You don't even need additional media queries to change the number of columns as the screen size changes. Just resize the window and see for yourself. The grid items will also automatically fill up the available space in the container. No complicated calculations required. Life changing.
